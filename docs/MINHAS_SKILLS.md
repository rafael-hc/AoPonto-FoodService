# 🤖 Guia de Capacidades da IA (Skills)

Este documento serve como uma referência rápida de todas as especialidades técnicas que eu posso aplicar no projeto **AoPonto**. Você pode me solicitar qualquer uma destas tarefas em português.

---

## 🎨 Frontend e Design System

### `frontend-blueprint` (Consultor de Design)
- **O que faz:** Guia a criação de interfaces de alta fidelidade antes de gerar código.
- **Use para:** "Projetar um novo dashboard", "criar um componente complexo", "melhorar a estética de uma página".
- **Dica:** Ideal quando você quer algo com visual premium (Linear/Vercel style).

### `react-composition-patterns` (Arquitetura React)
- **O que faz:** Aplica padrões avançados como *Compound Components* e *Render Props*.
- **Use para:** "Refatorar componentes com excesso de props", "criar uma biblioteca de componentes flexível".
- **Regra:** Segue as tendências do React 19.

### `react-best-practices` (Performance React)
- **O que faz:** Otimiza a renderização e o carregamento baseado nas diretrizes da Vercel.
- **Use para:** "Melhorar performance da página", "revisar hooks", "otimizar data fetching".

---

## 🏗️ Arquitetura e Backend (NestJS/DDD)

### `nestjs-modular-monolith` (Especialista em Monólitos Modulares)
- **O que faz:** Desenvolve backends escaláveis usando DDD, Clean Architecture e CQRS.
- **Use para:** "Criar um novo módulo de domínio", "definir limites de contextos (Bounded Contexts)", "implementar comunicação via eventos entre módulos".

### `domain-analysis` (Análise de Domínio/DDD)
- **O que faz:** Identifica fronteiras de negócio e sugere onde desenhar as divisões de serviços.
- **Use para:** "Quais domínios existem neste código?", "onde devo separar este serviço?".

### `coupling-analysis` (Análise de Acoplamento)
- **O que faz:** Mede quão dependentes os módulos estão entre si.
- **Use para:** "Estes módulos estão muito acoplados?", "relatório de qualidade de integração".

---

## ⚙️ Ferramental e Monorepo (Nx)

### `nx-workspace` & `nx-generate`
- **O que faz:** Configura o monorepo e gera novos projetos/bibliotecas/componentes seguindo os padrões do workspace.
- **Use para:** "Criar uma nova lib", "gerar um componente com Nx", "otimizar o cache do build".

### `nx-run-tasks`
- **O que faz:** Executa builds, testes e lints de forma eficiente (affected).
- **Use para:** "Rodar todos os testes", "verificar o que foi afetado pelo meu commit".

---

## 🛡️ Qualidade, Segurança e SEO

### `web-quality-audit` (Auditoria Completa)
- **O que faz:** Um review 360º cobrindo Performance, Acessibilidade, SEO e Boas Práticas.
- **Use para:** "Audite meu site", "minha página está boa para o usuário?".

### `security-best-practices`
- **O que faz:** Revisa o código em busca de vulnerabilidades (especialmente em TS/JS).
- **Use para:** "Fazer um review de segurança", "este código é seguro?".

### `accessibility` (A11y) & `seo`
- **O que faz:** Ajusta o código para leitores de tela e visibilidade em buscadores.
- **Use para:** "Melhorar acessibilidade", "otimizar meta tags para SEO".

---

## 🚀 Migração e Evolução

### `legacy-migration-planner`
- **O que faz:** Planeja a modernização de sistemas antigos (Legacy) para novas arquiteturas.
- **Use para:** "Como extrair este código antigo para um microserviço?", "planejar migração de framework".

### `component-identification-sizing`
- **O que faz:** Mapeia o tamanho dos componentes para priorizar o que precisa ser refatorado ou extraído.

## 🧠 Desenvolvimento de Skills

### `skill-architect`
- **O que faz:** Me ajuda a aprender novos fluxos de trabalho ou automatizar tarefas repetitivas criando novas skills.
- **Use para:** "Me ensine a fazer X de forma consistente", "crie um fluxo de trabalho automatizado para Y".
